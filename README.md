# Система управления BLDC

Репозиторий содержит полный набор исходников для двух контроллеров проекта:

1. **STM32G431 (B-G431B-ESC1)** – прошивка силовой части со встроенным FOC и обменом по UART.
2. **ESP32-S3 WROOM-1** – управляющая логика, веб-интерфейс, журналирование телеметрии и интеграция с датчиками.

Дополнительно включён минимальный HTML-интерфейс для локальной отладки.

## Структура

- `stm32_firmware/` – код на базе HAL/X-CUBE-MCSDK. Основные файлы размещены в `Core/Src` и `Core/Inc`.
- `esp32_firmware/` – проект ESP-IDF с разделением на модули (`button`, `state_machine`, `uart_comm`, `web_server` и т.д.).
- `web_interface/index.html` – упрощённая статическая страница, идентичная встроенному веб-интерфейсу.

## Сборка STM32

1. Создайте проект в STM32CubeIDE для платы B-G431B-ESC1, подключив X-CUBE-MCSDK.
2. Скопируйте файлы из `stm32_firmware/Core/Inc` и `stm32_firmware/Core/Src` в соответствующие каталоги проекта.
3. Проверьте пины силовой части и при необходимости скорректируйте определения в `main.c`.
4. Настройте измерительные каналы ADC и параметры MCSDK согласно схеме.

## Сборка ESP32-S3

```bash
cd esp32_firmware
idf.py set-target esp32s3
idf.py build
```

Для записи на модуль используйте USB-UART адаптер и команды `idf.py -p <порт> flash monitor`.

## Веб-интерфейс

- Тройное нажатие кнопки `PWR` включает точку доступа ESP32 (`BLDC-Setup / bldc-config`). Повторное тройное нажатие её выключает.
- Страница `/` предоставляет кнопки управления, настройку параметров механики и загрузку CSV-журнала.
- Логи сохраняются во внутренний SPIFFS (`/spiffs/session.csv`) и могут быть скачаны через HTTP.

## Журналирование

- Включается через веб-интерфейс или параметр `logging_enabled` (сохраняется в NVS).
- Формат CSV: `timestamp_ms,voltage_v,current_a,speed_rpm,temperature_c,motor_state,fault_mask,home,rew`.

## Защиты и логика

- Ограничение скорости по току реализовано на стороне STM32.
- ESP32 отслеживает сигналы HOME/REW, ток нагрузки и падение нагрузки для автоматического реверса.
- Любой переход направления инициируется последовательностью `стоп → пауза → старт в другую сторону`.

## Требования

- ESP-IDF 5.x или новее (для драйвера `led_strip`).
- STM32CubeIDE 1.10+ с пакетом MCSDK.

## Лицензия

Проект предназначен для демонстрации и дальнейшей адаптации под конкретный стенд.
